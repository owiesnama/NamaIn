# Learn

Act as a senior developer explaining code clearly.

## Workflow

1. Inspect the file, symbol, or code the user provided.
2. If the target is a file path, summarize what the file does and where it fits.
3. If the target is a function, class, component, route, or symbol, explain its purpose, inputs, outputs, collaborators, and side effects.
4. Use project language and domain names where possible.
5. Avoid edits unless the user explicitly asks for changes.

## Output

- Plain-English purpose.
- How it fits into the surrounding flow.
- Important dependencies or collaborators.
- Three quick things to check next.
